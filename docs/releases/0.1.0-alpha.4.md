# OfferLoop 0.1.0-alpha.4 发布说明

状态：预发布版本。日期：2026-07-23。

本次版本重新组织新用户文档，让安装、首次使用和后续增量更容易理解。四个 Skill 的业务能力、权限边界和数据合同没有变化。

## README 重构

- 按用户旅程重新组织 README：安装前准备、安装方法、四个 Skill、旧用户升级和其他说明。
- 在安装前准备中说明必需条件、检查方法，以及条件不满足时的解决办法。
- 提供两种通用安装入口：把 GitHub 链接交给支持标准 Skill 的 Agent，或使用终端安装命令。
- 删除针对具体 Agent 产品的逐项安装教学，统一使用标准 `SKILL.md` 和 Agent 自身的 Skills 目录表达。
- 分别说明四个 Skill 的作用、首次运行前准备、首次运行流程、首次输出、后续增量和案例。
- 明确 `offerloop-setup` 负责部署工作台和技术配置，`offerloop-workspace` 负责创建私有求职空间并接入已有工作台。
- 明确求职偏好由 `job-collection` 渐进询问，而不是由 `offerloop-setup` 收集。

## 安装

可以把仓库链接交给当前 Agent，请它安装 `skills/` 下的四个标准 Skill；也可以在终端运行：

```bash
npx skills add riwonswain-ovo/OfferLoop -g \
  -s offerloop-setup job-collection recruiting-reminder offerloop-workspace -y
```

安装完成后需要重新开启 Agent 会话，再调用 `offerloop-setup` 运行只读预检。

## 验收范围

- 文档契约测试验证 README 五个主要章节的顺序。
- 每个 Skill 都必须包含作用、首次准备、首次流程、首次输出、后续增量和案例。
- 安装与升级说明保持 Agent 中立，不包含特定 Agent 参数或产品专属教学。
- 原有安装器、冷安装、配置、同步和业务合同测试继续执行。

## 边界

这是 prerelease。本次没有新增业务功能，也不改变飞书、邮箱、日历、知识库或工作台的权限要求。文件安装和离线预检不代表线上能力已经可用，真实身份、scope、租户安装和资源权限仍需逐项核验。

# OfferLoop 0.1.0-alpha.3 发布说明

状态：预发布版本。日期：2026-07-23。

本次版本聚焦新用户第一次下载和安装 OfferLoop 时的可理解性、反馈准确性与安装速度，不改变飞书、邮箱、日历、知识库或工作台的授权边界。

## 安装体验修复

- README 顶部新增三步快速开始，明确 OfferLoop 会同时安装四个协作 Skill。
- 区分“Skill 文件已安装”“本机预检通过”和“线上能力可用”，避免把 `needs_action` 或 `unverified` 误判为安装失败。
- `--dry-run` 的普通终端输出明确显示未写入文件，并以 `would install`、`would upgrade` 描述计划动作。
- 安装成功后直接提示结束当前 Agent 会话、新开会话并调用 `offerloop-setup` 运行只读预检。
- 安装器在源校验、摘要计算与复制阶段直接剪枝 `node_modules`、`dist` 和 `build`。即使模板在本机执行过依赖安装或构建，也不会再因遍历生成文件而长时间无输出。
- 保留原有 JSON 状态、幂等安装、冲突不覆盖与升级前备份契约。

## 验收范围

- 仓库单元测试覆盖 dry-run 不写入、普通输出不误报成功、安装后提示和生成目录剪枝。
- 冷安装验收覆盖 Codex、Claude Code、Hermes 与腾讯 WorkBuddy 的四 Skill 安装、重复安装和 `collection` 首次预检。
- GitHub Actions 继续在 Ubuntu、macOS 和 Windows 执行冷安装，并在 Python 3.10/3.12 下运行仓库合同测试。
- 两份妙搭模板继续执行锁定依赖安装、测试、类型检查和构建。

## 安装

```bash
git clone https://github.com/riwonswain-ovo/OfferLoop.git
cd OfferLoop
python3 scripts/install_offerloop.py --agent codex
```

将 `codex` 替换为 `claude-code`、`hermes-agent` 或 `workbuddy`。安装后结束当前 Agent 会话并新开会话，再调用 `offerloop-setup` 运行只读预检。

## 边界

这是 prerelease。文件安装和离线预检不验证飞书身份、scope、应用版本发布、租户安装、资源共享、邮箱授权、日历授权或工作台 OAuth；这些条件仍须按用户与资源逐项核验。

# OfferLoop 0.1.0-alpha.2 发布准备

状态：候选版本，尚未创建 Git tag 或 GitHub Release。日期：2026-07-23。

本次版本聚焦新用户在腾讯 WorkBuddy 上安装后首次运行 `offerloop-setup` 时可能遇到的本地依赖卡点；不改变飞书、邮箱、日历、知识库或工作台的授权边界。

## 包含的修复

- WorkBuddy 使用其自身管理的 `lark-cli`。当该副本低于 OfferLoop 所需的 `1.0.73` 时，预检现在会识别这一情况，并提供只更新 WorkBuddy 管理前缀中官方 CLI 的恢复动作。
- 恢复动作不会复制可执行文件、删除 profile 或写入 OfferLoop 业务配置；完成后须重启 WorkBuddy，并在新任务中重新执行只读预检。
- 已在真实 WorkBuddy 5.1.7 新任务中复验：管理副本升级到 `lark-cli 1.0.76` 且可发现 profile 后，`collection` 本地检查没有 `blocked` 或 `needs_action`；线上权限和通知继续按设计显示为 `unverified`。
- Codex、Claude Code、Hermes 和 WorkBuddy 仍统一安装四个公共 Skill；不引入第五套业务指令或专属业务副本。

## 升级方式

在仓库根目录执行（将 Agent 名替换为实际使用的 Agent）：

```bash
git pull --ff-only
python3 scripts/install_offerloop.py --agent workbuddy --upgrade
```

安装器仅备份并替换 OfferLoop Skill 文件；不会删除 `~/.config/offerloop/`、`~/.local/state/offerloop/`、飞书 profile 或凭证。安装后结束当前 Agent 会话并新开会话。

如 WorkBuddy 的只读预检提示其管理的飞书 CLI 版本过低，请仅在确认提示指向 WorkBuddy 管理副本后执行预检展示的恢复动作；不要把 profile、App Secret、密码、Cookie 或 token 发送到聊天中。

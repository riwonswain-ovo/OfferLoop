# OfferLoop 使用指南

OfferLoop 把招聘信息、真实投递、笔试面试和求职材料组织在同一个私有求职空间中。
三张 Base 是唯一数据源；本首页只负责说明和导航，不复制实时业务数据。

## 从这里开始

[打开 OfferLoop 招聘工作台]({{workbench_url}})

工作台集中展示招聘企业、求职进展和笔面试安排。需要编辑完整记录时，从工作台进入对应 Base。

## 第一次使用

1. 运行 `offerloop-setup`，选择当前要启用的能力并完成预检。
2. 在企业清单中筛选岗位，将真实投递的记录改为“已投递”。
3. 在求职进展中手动补充投递岗位、岗位 JD 和投递日期。
4. 启用邮件提醒后，确认识别结果和日历方案，再写入笔面试中心与个人日历。

## 日常流程

1. 用 `job-collection` 同步招聘信息。
2. 在工作台查看企业清单，并维护真实投递状态。
3. 用 `recruiting-reminder` 整理笔试面试邮件。
4. 在知识库目录中归档简历、面试准备和面试复盘文档。

## 常见问题

### 为什么首页不展示实时表格和日历？

实时数据统一由招聘工作台和三张 Base 展示，避免首页产生副本、过期内容或重复维护。

### 数据应该在哪里修改？

招聘信息在求职企业清单维护，真实申请在求职进展维护，笔试面试安排在笔面试中心维护。

### Skill 更新会覆盖我的材料吗？

不会。个人材料和业务数据留在你的飞书空间；Skill 只保存资源入口和必要的本地定位信息。

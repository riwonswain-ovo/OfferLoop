# OfferLoop 使用指南与目录契约

## 首页定位

首页固定为 `00｜OfferLoop 使用指南`，服务于新手指引、功能说明和导航，不承担实时业务数据展示。第一操作入口必须是 `01｜求职工作台`。工作台负责查看实时数据；三张 Base 是唯一数据源。

首页应包含：

- 从这里开始：工作台链接与一句职责说明。
- 第一次使用清单与日常流程。
- 核心功能和数据位置说明。
- 可对 Codex 使用的自然语言命令；未实现能力须标注“即将支持”。
- 数据不重复维护原则与常见问题。

## 固定目录

```text
00｜OfferLoop 使用指南
01｜求职工作台
02｜求职业务 / 求职企业清单、求职进展、笔面试中心
03｜个人材料 / 当前简历、历史简历、作品集、其他求职材料
04｜面试准备 / 待准备、准备完成、通用准备材料
05｜面试复盘 / 待复盘、已完成复盘
06｜训练与题库 / 简历深挖、产品 Sense、通用面试题、笔试资料
07｜信息源 / 招聘信息源、公司招聘入口、求职工具
99｜归档 / 历史求职材料、结束流程文档
```

## 数据与维护边界

- 求职企业清单、求职进展和笔面试中心不可复制到知识库文档；知识库只登记其入口与用途。
- `job-collection` 只更新招聘信息；`recruiting-reminder` 只更新笔面试中心和求职进展。
- 面试准备和复盘文档推荐命名为 `公司｜岗位｜环节准备｜YYYY-MM-DD`、`公司｜岗位｜环节复盘｜YYYY-MM-DD`。
- 不按公司新建顶级目录；所有新增材料必须归入现有目录。
- 自动化不得改写首页的说明文字，除非用户明确要求重构首页。
